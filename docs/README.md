# 面试复习文档索引

> 这套文档是为「拿 English-AI-tutor 项目去面试」准备的。目标是：面试前能快速扫读复习，面试时能清晰、有条理地讲清楚「这个项目是什么、用了什么技术、为什么这么选、遇到过什么难点、怎么解决的」。

## 文档清单与用途

| 文档 | 用途 | 什么时候看 |
|------|------|-----------|
| [01-项目概览.md](./01-项目概览.md) | 一句话讲清项目是什么、有哪些角色和功能、整体架构长什么样 | 面试开场「介绍一下你的项目」 |
| [02-技术栈与选型理由.md](./02-技术栈与选型理由.md) | 完整技术栈清单 + 每个关键技术「为什么选它、对比了什么、取舍是什么」 | 被追问「为什么用 X 不用 Y」 |
| [03-系统架构与数据流.md](./03-系统架构与数据流.md) | 架构分层图、认证流程、预约状态机、实时通知数据流、AI 匹配流程等 | 被要求「画一下流程 / 讲一下某个功能怎么实现」 |
| [04-难点与解决方案.md](./04-难点与解决方案.md) | 用 STAR 结构（情境/任务/行动/结果）讲 9 个技术难点 | 被问「你遇到过什么难点、怎么解决的」 |
| [05-高频面试问答.md](./05-高频面试问答.md) | 按主题分类的预设问答，每题给「考点 + 参考答案要点」 | 面试前一晚集中过一遍 |

## 建议复习顺序

1. **先通读 `01`**：确保能在 1 分钟内讲清项目定位和规模。
2. **再读 `03`**：把核心流程的「数据怎么流」在脑子里跑通，这是技术面的主战场。
3. **重点啃 `04`**：难点是面试的「亮点输出」，每个难点要能脱稿讲出 STAR 四段。
4. **配合 `02`**：选型理由是「为什么」类问题的弹药库。
5. **最后刷 `05`**：模拟问答，查漏补缺。

## 项目一句话定位

> 一个英语 AI 家教平台：学生可以浏览/AI 智能匹配外教、预约试听课；外教管理可约时段和预约请求；管理员审核外教入驻。前后端分离，含完整的认证、预约状态机、实时通知、后台定时任务和 AI 匹配能力。

## 关于真实性的说明

- 文档中所有「已实现」的技术细节，均对应仓库里的真实代码，并标注了文件路径，方便面试前对着代码再过一遍。
- 凡是标注「**可进一步优化 / 延伸思考**」的内容，是该技术栈下合理的改进方向，**项目中尚未实现**。面试时如果讲到，要明确说「这是我后续会做的优化」，不要说成已完成，避免被追问时露馅。
