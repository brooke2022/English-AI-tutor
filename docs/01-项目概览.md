# 01 · 项目概览

> 这份文档帮你应对面试开场的「先介绍一下你的项目」。目标：30 秒讲清定位，2 分钟讲清功能和架构。

---

## 1. 一句话介绍

**English-AI-tutor 是一个英语在线家教平台**，连接想学英语的学生和全球外教。核心特色是用 AI（Google Gemini）根据学生的学习目标智能匹配最合适的外教，并提供完整的预约、上课、评价闭环。

项目是**前后端分离的全栈应用**：
- 前端：React 19 单页应用（SPA）
- 后端：NestJS 提供 RESTful API + WebSocket 实时通信
- 部署：前后端各自 Docker 化，部署到 Fly.io

---

## 2. 项目背景（为什么做这个项目）

可以这样向面试官表述（按你的真实情况调整）：

- **业务出发点**：在线 1v1 外教是个成熟赛道（VIPKID、italki 等），但选老师对学生是个痛点——面对几百个老师不知道选谁。我想做一个「用 AI 降低选择成本」的版本。
- **技术出发点**：想做一个**功能完整、能讲出深度**的全栈项目，而不是 CRUD demo。所以我特意覆盖了几个有技术含量的点：带状态机的预约系统、JWT 双 token 认证、WebSocket 实时通知、后台定时任务、第三方 AI 集成、容器化部署。
- **角色复杂度**：平台有学生、外教、管理员三种角色，天然需要权限控制（RBAC），适合体现工程能力。

---

## 3. 三种角色与核心功能

平台围绕 **学生 / 外教 / 管理员** 三种角色组织。

### 学生（STUDENT）
- 注册/登录，浏览已审核通过的外教列表（支持按关键词、技能标签、国家筛选）
- 查看外教详情：介绍视频、评价、可预约时段
- **AI 智能匹配**：输入学习目标（如 "I want to improve speaking for job interviews"），AI 推荐最多 3 位外教并给出推荐理由
- 预约外教某个时段的课（生成 PENDING 预约）
- 在「我的课程」查看预约状态，课程确认后进入教室、取消预约
- 课程完成后对外教评分（1-5 星 + 评论）

### 外教（TEACHER）
- 注册时填写国家、自我介绍、技能标签、定价等（注册后状态为 PENDING，需管理员审核）
- 设置每周可约时段（WeeklySlot 周模板）
- 在「预约管理」查看待处理的预约请求，**接受**（填会议链接）或**拒绝**（填理由）
- 仪表盘查看统计数据、今日课程

### 管理员（ADMIN）
- 审核外教入驻：查看 PENDING 外教资料，**批准**或**拒绝**
- 管理员不能自助注册（后端硬性拦截），只能由种子数据/手动创建

### 全局能力
- **实时通知**：预约创建/确认/取消、课程提醒，通过 WebSocket 实时推送，导航栏通知铃实时更新未读数
- **国际化**：中英文切换（i18next），自动检测浏览器语言并记忆用户选择

---

## 4. 关键业务流程（一句话串起来）

```
注册/登录 → 浏览外教 或 AI 匹配 → 选时段预约(PENDING)
   → 外教接受(CONFIRMED, 带会议链接) 或 拒绝(CANCELLED)
   → 到点上课 → 课后 60 分钟系统自动标记 COMPLETED
   → 学生评价 → 外教评分均值更新
```

每一步状态变化都会触发**邮件 + 站内实时通知**给相关方。

## 5. 项目规模（面试时用来量化表述）

| 维度 | 数据 |
|------|------|
| 后端业务模块 | 13 个 NestJS Module（auth / users / teachers / bookings / reviews / notifications / jobs / ai / uploads / realtime / admin / prisma / health） |
| API 端点 | 约 25 个 RESTful 接口，统一前缀 `/api/v1` |
| 数据表 | 10 张（User / StudentProfile / TeacherProfile / TeacherTag / WeeklySlot / Booking / Review / Favorite / RefreshToken / Notification） |
| 用户角色 | 3 种（STUDENT / TEACHER / ADMIN），RBAC 权限控制 |
| 前端页面 | 16 个路由页面（公开页 + 学生/外教/管理员各自的工作台） |
| 状态机 | 预约 4 状态（PENDING / CONFIRMED / COMPLETED / CANCELLED）、外教 3 状态（PENDING / APPROVED / REJECTED） |
| 第三方集成 | Google Gemini（AI 匹配）、Resend（邮件）、AWS S3/R2（文件存储）、Redis（任务队列） |

**讲解要点**：
- 前后端**完全分离**，前端是纯静态资源（Nginx 托管），后端是独立 API 服务，两者各自容器化、各自部署。
- 后端按**业务领域模块化**（NestJS Module），每个模块内部是 Controller → Service → (Prisma) 的分层。
- 所有外部依赖（Redis / S3 / Gemini / Resend）都是**可选的**——没配置时后端自动降级，核心流程不受影响（详见 [04-难点与解决方案](./04-难点与解决方案.md) 难点 5）。

## 7. 前端目录结构

```
src/
├── main.tsx / App.tsx        # 入口 + 路由表
├── pages/                    # 16 个路由页面
│   ├── auth/                 # 登录、注册（学生/外教）
│   ├── student/              # 学生工作台、个人资料
│   ├── teacher/              # 外教工作台、可约时段、预约管理、资料
│   └── admin/                # 管理员审核台
├── components/               # 复用组件（Navbar / ProtectedRoute / TeacherCard / NotificationBell ...）
├── hooks/                    # 数据获取层：基于 React Query 封装的业务 hooks
├── lib/                      # 基础设施：api(axios) / queryClient / socket
├── store/                    # Zustand：仅 useAuthStore（认证状态）
├── utils/                    # 纯函数工具（时间格式化等）
├── types/                    # TypeScript 类型定义
└── locales/                  # i18n 中英文翻译资源
```

**讲解要点**：`hooks/` 和 `lib/` 是这个项目前端架构的核心——`lib/` 放基础设施（HTTP 客户端、缓存配置、WebSocket），`hooks/` 把每个业务的数据获取逻辑封装成自定义 hook，页面组件只调 hook、不碰 API 细节。

## 9. 一段话「项目自我介绍」模板（可背）

> 「我做了一个英语 AI 家教平台，叫 English-AI-tutor，是个前后端分离的全栈项目。前端用 React 19 + Vite + TypeScript，后端用 NestJS + Prisma + PostgreSQL。它有学生、外教、管理员三种角色，核心功能是学生可以用 AI 智能匹配外教、预约试听课，外教管理自己的可约时段和预约请求。
>
> 我在这个项目里重点打磨了几个有技术含量的点：一个是带状态机的预约系统，用数据库唯一约束解决了并发重复预约的问题；一个是 JWT 双 token 认证 + axios 拦截器实现的无感刷新；还有 WebSocket 实时通知、用 BullMQ 做的课程提醒定时任务、以及接入 Google Gemini 做 AI 匹配。整个项目前后端都做了 Docker 容器化，部署在 Fly.io 上，配了 GitHub Actions 做 CI。」

（讲完这段，面试官通常会挑一个点深入问，对应 [03](./03-系统架构与数据流.md) 和 [04](./04-难点与解决方案.md)。）
